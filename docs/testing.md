# 测试与验收

本文档说明这个仓库如何验证“能跑、可回归、边界明确”。

重点不是证明它已经是完整商业产品，而是确认当前这版原型在核心链路上可重复、可解释、可维护。

---

## 1. 测试目标

当前测试主要覆盖四类风险：

1. **时间修正是否正确**
   - 真太阳时是否同时考虑经度修正与时间方程
   - 子时边界是否正确区分早子时 / 晚子时

2. **地点解析是否可靠**
   - 常见区县级地点能否命中
   - 别名是否能解析
   - 歧义地点是否会显式报错，而不是静默猜测

3. **排盘主链路是否可用**
   - 从输入参数到 `iztro` 排盘结果的整条链路是否正常
   - 经纬度直传模式是否可用

4. **CLI 对外行为是否稳定**
   - 文本输出能否正常生成
   - JSON 输出能否正常生成
   - 错误场景是否返回非零退出码并输出可识别报错

---

## 2. 测试分层

### 单元测试

覆盖底层规则和边界行为：

- 真太阳时计算
- 子时索引映射
- 地理解析
- 地址歧义处理

这类测试用于尽早发现“逻辑改了一点点，但盘面基础已偏掉”的问题。

### 集成测试

覆盖从用户输入到排盘结果的完整主链路：

- 区县级地点输入
- 经纬度直传输入
- `buildChart` 的整体返回结构

这类测试用于确认模块之间能协同工作，而不是只在局部函数层面正确。

### 回归测试

使用固定出生样例校验关键字段不漂移。

它的价值在于：当你重构时间修正、地址解析或输出结构时，能快速判断是否对既有样例造成了非预期影响。

### CLI 验收测试

直接从命令行执行脚本，验证面向外部调用方的行为：

- 文本输出可生成
- JSON 输出可生成
- 歧义场景会失败并给出明确错误提示

---

## 3. 本地执行

先安装依赖：

```bash
npm install
```

再执行自动化测试：

```bash
npm test
npm run test:cli
```

如需人工抽查，可再执行以下样例：

```bash
npm run chart -- --date 1990-05-15 --time 14:30 --gender male --place 北京市东城区
npm run chart -- --date 1988-02-09 --time 23:40 --gender female --place 杭州市余杭区 --json
npm run chart -- --date 1990-05-15 --time 14:30 --gender male --place 和平区
```

建议把第三条也跑一遍，确认歧义地名不会被默默解析成某一个行政区。

---

## 4. 当前验收口径

以下条目可视为当前版本“可交付原型”的最低验收标准：

- [x] 能接收 `date + time + gender + place` 进行排盘
- [x] 能按区县级地点解析经纬度
- [x] 能接受 `longitude + latitude` 直传模式
- [x] 地点歧义时会提示候选，不默默猜测
- [x] 真太阳时同时考虑经度修正与时间方程
- [x] 早子时 / 晚子时正确区分为 `timeIndex=0/12`
- [x] 能输出文本报告
- [x] 能输出 JSON 结构化结果
- [x] OpenClaw skill 调用入口保持可用
- [x] 本地自动化测试可完整跑通

---

## 5. 当前不在验收范围内

以下事项目前不应作为“测试失败”判断依据，因为它们本就不是当前版本承诺的范围：

- 全国全量区县覆盖率
- 海外地址解析覆盖率
- 多流派命理解释一致性比对
- 自动生成完整长篇咨询报告
- Web API、数据库、鉴权、任务队列等服务端能力

把这些内容写清楚，有助于避免文档口径和实际实现错位。

---

## 6. 当前数据覆盖说明

当前地理数据为最小可运行集：

- 覆盖全国 34 个省级行政区的代表性区县
- 额外补充若干高频出生地
- 支持别名匹配与歧义检测
- 仍不是全国全量区县库

因此，测试通过代表“当前样例与已收录地点可稳定工作”，不代表“任何中国地名输入都一定能直接命中”。

---

## 7. 建议的发布前检查

如果要把当前版本作为 demo、PoC 或内部交付件展示，建议至少执行：

1. `npm test`
2. `npm run test:cli`
3. 人工跑 1 条文本输出样例
4. 人工跑 1 条 JSON 输出样例
5. 人工跑 1 条歧义地点样例

这样可以同时覆盖：

- 内部逻辑
- 外部 CLI 入口
- 成功路径
- 失败路径

对于当前阶段，这已经足够支撑“可演示、可集成、可复现”的质量要求。
