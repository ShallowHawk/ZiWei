# 使用限制与风险提示

本文档用于说明这个仓库**能做什么、不能做什么、哪些地方容易误导用户**。

## 1. 定位

本项目当前定位是：

- 紫微斗数排盘原型
- 面向 Agent / CLI / 工作流集成的结构化输出工具
- 传统命理研究与娱乐体验用的工程基座

它**不是**：

- 医疗、法律、投资、婚育等重大决策工具
- 专业命理咨询服务替代品
- 对人生结果作确定性判断的预测系统
- 已覆盖全国任意地点、任意流派、任意口径的完整产品

## 2. 容易误导用户的风险点

### 2.1 把“排盘结果”误当成“客观事实”

本仓库输出的是基于既定规则、地理坐标与时间修正后的盘面结果，属于**传统命理框架下的解释性素材**，不等同于可验证的客观事实。

因此：

- 不应把文本摘要写成“必然发生”“注定如此”
- 不应把宫位/星曜组合直接表述为确定的健康、婚姻、财富结论
- 不应把命盘输出包装成对未来的保证或承诺

### 2.2 把“结构化 highlights”误当成最终解读

`highlights` 只是重点宫位的结构化摘要，方便上层模型继续组织内容，**不是完整结论**。

如果直接把这些字段扩写成长文，存在以下风险：

- 脱离盘面上下文过度发挥
- 忽略三方四正、四化、门派差异
- 对敏感议题生成过度确定的表述

因此，上层模型只能将其视为“待解释素材”，不能视为“已定稿判断”。

### 2.3 出生时间误差会放大结论偏差

真太阳时修正和子时边界会影响 `timeIndex`，从而改变盘面。

高风险场景包括：

- 用户只记得大概时段，不记得准确分钟
- 用户报的是口述时间、户口登记时间、医院记录时间中的某一种，但来源不明
- 出生时间靠近整点、尤其接近 `23:00 / 00:00 / 01:00`
- 出生地只给到省/市，未精确到区县，导致经度修正偏差变大

因此，在接近时辰边界时，应明确告知用户：**几分钟误差就可能导致盘面变化**。

### 2.4 地点解析并非无限覆盖

虽然仓库已支持较完整的中国区县数据与歧义提示，但仍有边界：

- 海外地点不保证稳定解析
- 历史区划、撤县设区、旧地名别称不保证全覆盖
- 用户输入过于模糊时，系统会拒绝猜测，但也可能无法直接命中

“没命中地点”不应被误解为“地点不存在”，而更可能是：

- 输入不够精确
- 数据未收录
- 历史地名尚未映射

### 2.5 理论口径存在门派差异

本仓库当前重点是**稳定排盘**，不是统一所有命理流派的解释分歧。

因此：

- 不保证不同门派对同一盘面的结论一致
- 不保证自动生成的自然语言报告符合某一门派的严格口径
- 不应宣称为“权威定论”或“专业咨询结论”

## 3. 对外表述建议

推荐用语：

- “排盘结果 / 盘面摘要 / 结构化结果”
- “供研究、参考或娱乐体验使用”
- “若需进一步解读，应结合更多上下文谨慎分析”
- “敏感议题请以专业机构意见为准”

避免用语：

- “准确预测你的人生结果”
- “自动给出最终结论”
- “保证婚姻/财富/健康判断正确”
- “替代医生、律师、理财顾问、心理咨询师”

## 4. 建议的上层交付规则

如果要把本仓库接到聊天机器人、Agent 或报告生成流程中，建议遵循：

1. 先确认出生日期、时间、性别、地点是否足够精确
2. 对模糊地点或边界时辰继续追问，不要替用户脑补
3. 对健康、寿命、疾病、法律纠纷、投资收益等议题加显式免责声明
4. 避免使用宿命化、恐吓式、单结论式表达
5. 把输出定位为“传统命理视角下的参考性分析”，而不是事实认定

## 5. 当前仓库已采取的收紧措施

- 地点歧义时明确报错，不静默猜测
- 文本报告加入用途限制与风险提示
- JSON 输出补充 `usageNotice` 与 `limitations`
- README 与 skill 文档明确链接到本限制说明

## 6. 一句话结论

这个仓库当前更适合被描述为：

**一个带真太阳时修正与地点解析的紫微斗数排盘/结构化输出原型，而不是可对人生重大事项下确定结论的自动化咨询系统。**
